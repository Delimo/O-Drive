# 后台管理 Tab 重做方案

> **改动范围说明**：本方案仅改动后台管理页 tab 切换后**内容框（explorer-card）内的显示效果**，即下图标注的区域。以下内容均不涉及改动：
> - 网站全局布局（顶栏、侧边栏、页脚等）
> - Tab 栏本身的样式与交互
> - 文件管理页（`index.html`）、分享页（`share.html`）等非后台页面
> - 后端 API 逻辑（除非必要的新增字段）
>
> 简单说：**只改每个 tab 点开后里面展示什么、怎么排，其他一切不动。**

## 现状问题

| 问题 | 说明 |
|------|------|
| Tab 过多 | 9 个 tab，顶栏快塞不下，小屏更严重 |
| 内容重叠 | 「系统状态」里包含维护操作，「维护」tab 又有同样的维护操作，用户混淆 |
| 部分 tab 内容薄弱 | 「通知」仅一张小表，独占一个 tab |
| 缺少功能入口 | 回收站保留天数设置、S3 存储配额编辑等无 UI |
| 返回主页无入口 | 登录管理员后「返回云盘」按钮消失 |

│overview│ storage│ shares │ paths  │  logs  │maint.  │ system │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┘

### 1. 概览 `overview`

**来源**: 现有「概览」不变

**内容**:
- 文件总数卡片（数量、总容量、文件夹数）
- 回收站项目卡片（数量、占用、建议清理标签）
- 索引状态卡片（索引条数、推荐状态）
- 类型分布网格（各文件类型计数 + 大小）
- 系统提醒列表
- 最近资源列表

**改进点**:
- 最近资源的 `latest-chip-name` 改为展示文件名（`item.name`）而非完整路径（`item.key`）
- 索引状态卡片添加「立即重建」快捷按钮

---

### 2. 存储 `storage`

**来源**: 现有「存储」+ 回收站保留天数设置

**内容**:

**2.1 Cloudflare R2**
- 使用量进度条 + 配额编辑按钮

**2.2 S3 存储空间**
- 空间列表（名称、桶、端点、已用/配额、状态）
- 添加/测试/删除按钮

**2.3 路径绑定**
- 绑定列表（路径 → 存储空间映射），添加/删除

**2.4 溢出策略**
- 启用/禁用开关 + 阈值设置

**2.5 回收站保留天数** ← **新加**
- 数字输入 + 保存按钮（默认 0 = 不自动清理）
- 说明文案："超过保留天数的回收站文件将在执行清理时自动删除"

**涉及新增后端调用**:
- `GET|PUT /api/admin/settings/trash-retention`（已有，缺 UI）
- `POST /api/trash/cleanup`（已有，缺 UI，可由维护 tab 触发）

---

### 3. 分享 `shares`

**来源**: 现有「分享」不变

**内容**:
- 统计栏（总数、有效、已失效）
- 筛选标签（全部/有效/已过期/次数用尽）
- 分享列表（名称、token、链接、到期时间、下载次数、状态标签）
- 复制链接、删除

**改进点**:
- 搜索框：按文件名/token 搜索
- 批量操作：勾选 + 「批量删除」「清理过期」
- 分页：分享较多时分页加载

---

### 4. 路径 `paths`

**来源**: 现有「路径管理」

**内容**:

**4.1 受保护路径**
- 各自独立的「添加」按钮（不再共用顶栏一个按钮）
- 列表：路径名、备注、显示名称、删除

**4.2 隐藏路径**
- 各自独立的「添加」按钮
- 列表：路径、取消隐藏

**改进点**:
- 两个区块各自有独立的添加按钮，消除歧义
- 受保护路径可内联编辑备注

---

### 5. 日志 `logs`

**来源**: 现有「日志」

**内容**:
- 筛选栏：关键词 + 操作类型（创建/删除/更新/分享/登录/上传）+ **日期范围** ← 新增
- 日志列表（操作、路径、用户、IP、时间、详情）
- 分页

**改进点**:
- 添加 `data-action="set-logs-date-range"` 日期范围选择器
- 支持导出当前筛选结果为 CSV

---

### 6. 维护 `maintenance`

**来源**: 现有「维护」+ 从「系统状态」移过来的维护操作

**内容**:

**6.1 维护快照**
- 索引记录数、R2 对象数、访问失败记录数
- 回收站项目数、操作日志数、后台任务数、缩略图缓存状态

**6.2 维护操作卡片**
- 重建文件索引
- 清理访问记录
- 清理缩略图缓存
- 清理旧操作日志
- 清理已完成任务
- 确认系统提醒
- **清理过期回收站** ← 新增（调用 `POST /api/trash/cleanup`）

**6.3 后台任务表**
- 文件数、进度、状态、时间

**说明**: 将「系统状态」中的维护快照 + 操作按钮全部移到这里，系统 tab 不再包含维护内容。

---

### 7. 系统 `system`

**来源**: 现有「系统状态」减去维护部分 + 现有「通知」+ 现有「Webhook」

**内容**:

**7.1 环境检查**
- 各组件（DB、R2、环境变量等）健康状态

**7.2 存储配额**
- 当前用量 / 总量百分比进度条

**7.3 通知列表** ← 从独立 tab 移入
- 时间、消息、已读/未读、标记已读按钮
- 全部标记已读

**7.4 Webhook 配置** ← 从独立 tab 移入，折叠区域
- Webhook 列表（名称、URL、事件、状态），折叠展开
- 添加/编辑/测试/删除
- 投递记录（折叠展开）

**说明**: 「通知」和「Webhook」都是系统级别的配置与查看，合入「系统」后释放了两个 tab 位置，且内容更充实。Webhook 投递记录默认折叠，减少视觉噪音。

## 实施优先级建议

| 优先级 | 步骤 | 预估工作量 | 说明 |
|--------|------|-----------|------|
| **P0** | `ADMIN_TABS` 改为 7 个 + 路由分发调整 | 小 | 基础骨架，必须先做 |
| **P0** | 管理员登录后显示「返回云盘」链接 | 极小 | 行级修改 |
| **P1** | 重写「系统」tab（环境 + 配额 + 通知 + Webhook 折叠） | 中 | 合并三个旧 tab，注意折叠状态管理 |
| **P1** | 重写「维护」tab（加入清理过期回收站） | 中 | 新增操作按钮 + thunk |
| **P1** | 「存储」tab 加入回收站保留天数设置 | 小 | 接入已有 API |
| **P2** | 「路径」tab 独立添加按钮 | 小 | 改 render 和事件 |
| **P2** | 「日志」tab 日期筛选 + CSV 导出 | 中 | 前端日期组件 + 后端需支持可选 date 参数 |
| **P3** | 「分享」tab 搜索 + 批量操作 + 分页 | 大 | 涉及勾选状态管理、批量 API 调用 |

## 风险与注意事项

1. **「系统」tab 内容过长** — 环境 + 配额 + 通知 + Webhook 四个区块纵向堆叠，在内容丰富的场景下可能一屏看不完。建议：
   - Webhook 区块默认折叠（用 `<details>` 或自定义折叠组件）
   - 通知列表默认只显示最近 10 条，提供「查看全部」展开

2. **通知和 Webhook 合入后的状态管理** — 目前 thunk 和 action 是独立的（`loadAdminNotifications` / `loadAdminWebhooks`），迁移后需要确保切入「系统」tab 时**同时加载**这些数据，避免因数据未加载而显示空白。

3. **日期筛选对后端的影响** — 日志 tab 的日期范围筛选需要后端 `GET /api/admin/logs` 支持可选的 `startDate` / `endDate` 参数。当前后端可能不支持，需确认后再实施。

4. **分享 tab 批量操作的交互复杂度** — 勾选状态需要在 store 中新增 `admin.selectedShareTokens: string[]`，且要处理「全选」「取消全选」「跨页选择」等边界情况，建议放在最后做。

5. **回收站保留天数与清理的联动** — 用户在「存储」tab 设置了保留天数后，应该能在「维护」tab 手动触发清理，或提示「已设置自动保留 X 天」。两处需要同一数据源，建议将 `trashRetentionDays` 存入 `admin` state。

6. **测试回归** — 合并 tab 后现有 Playwright 测试（`tests/browser/admin-flow.spec.mjs`）中的定位器可能失效，需要在实施后更新测试用例。
