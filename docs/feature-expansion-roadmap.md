# O-Drive 功能扩展路线图

> 更新时间：2026-06-28
> 目标：记录仍值得继续推进的功能方向，避免把已经实现的能力重复列为待办。

## 当前状态总览

| 功能 | 当前状态 | 说明 |
| --- | --- | --- |
| 文件夹分享 | 已实现 | 已支持 `target_type`、目录浏览、目录内文件下载/预览和目录 ZIP 下载。 |
| 批量/打包下载 | 已实现 | 小目录即时下载，大目录会自动转后台 `zip_download` 任务，完成后可从任务列表或通知取回 ZIP。 |
| 回收站增强 | 部分完成 | 已支持恢复、永久删除、清空、保留天数、筛选；还可增强冲突处理和批量体验。 |
| 搜索增强 | 部分完成 | 已支持 D1 文件索引、名称搜索、类型/大小/时间等过滤；全文搜索未实现。 |
| 通知与告警中心 | 部分完成 | 已有站内通知、Webhook 配置、投递记录、失败重试、容量阈值告警和失败任务数量告警；下载异常频率规则和通知筛选仍可增强。 |

## 推荐优先级

1. 回收站冲突处理和批量体验
2. 搜索能力增强，尤其是全文搜索
3. 告警规则剩余项和通知筛选

## 1. 文件夹分享

### 目标

让分享对象支持文件和目录。目录分享页可以逐级浏览，也可以打包下载目录内容。

### 当前状态

- `share_links` 已增加 `target_type` 字段。
- 分享创建会自动识别文件或目录。
- 目录分享页支持当前目录浏览、子目录跳转、目录内文件预览/下载。
- 目录下载会生成 ZIP 响应。

### 已落地方案

- 数据层使用 `target_type: file | folder`，并保留 `path` 作为目标路径。
- 分享创建时根据路径判断文件或目录。
- 目录分享访问接口返回当前目录子项和权限信息。
- 目录下载复用 ZIP 流式打包能力。

### 验收标准

- 可以创建目录分享链接。
- 访问目录分享链接时可以浏览目录内文件。
- 可以下载目录中的单个文件。
- 可以对整个目录发起打包下载。
- 过期、密码、下载次数限制对文件和目录都生效。

## 2. 大目录打包下载体验优化

### 当前状态

基础打包下载和大目录后台化已经存在：小目录仍直接流式返回 ZIP，大目录会创建后台任务，任务完成后写入通知并提供结果下载链接。

### 已落地方案

- 对目录体量做预估，超过阈值时自动转后台任务。
- 后台任务复用 `file_tasks`，类型为 `zip_download`。
- ZIP 结果写入 `.system/zip-tasks/...`，任务结果包含下载地址。
- 任务完成后写入通知中心，用户可以从通知和任务列表取回结果。

### 后续可增强

- 增加失败任务重试入口。
- 对超大目录增加最大文件数、最大总大小等硬限制。
- 对后台 ZIP 结果增加自动清理策略。

## 3. 回收站增强

### 当前状态

已经支持：

- 批量删除进入回收站。
- 恢复回收站记录。
- 永久删除回收站记录。
- 清空回收站。
- 按保留天数清理。
- 按路径、类型、删除时间筛选。

### 建议增强

- 恢复时遇到同名冲突，允许选择跳过、覆盖或自动重命名。
- 支持批量恢复前预估冲突数量。
- 回收站列表增加更明确的批量操作反馈。
- 管理后台展示预计可清理容量。

## 4. 搜索增强

### 当前状态

已经支持：

- D1 `file_index` 元数据索引。
- 名称搜索。
- scope 范围搜索。
- 类型、大小、修改时间过滤。
- 搜索分页/游标。

### 建议增强

- 增加全文索引表或外部全文检索方案。
- 支持文本、Markdown、代码文件内容检索。
- 搜索结果展示命中原因，例如文件名命中、路径命中、内容命中。
- 管理后台增加索引健康状态和重建入口。

## 5. 通知与告警中心

### 当前状态

已经支持：

- 站内通知表。
- 通知已读状态。
- Webhook 配置。
- Webhook 投递记录。
- 下载异常和分享过期等事件通知基础。
- Webhook 失败投递支持手动重试。
- 容量阈值告警支持 warning/error 百分比配置，并写入站内通知。
- 失败任务数量告警支持统计窗口、warning/error 条数配置，并写入站内通知。

### 建议增强

- 增加下载异常频率等剩余可配置告警规则。
- 抽出通用告警规则 helper，统一阈值判断、冷却和通知写入。
- 通知增加 severity：info、warning、error。
- 管理后台支持按事件类型和严重程度筛选通知。

## 工程建议

- 新能力优先复用现有 `tasks`、`notifications`、`share_links`、`trash`、`file_index`。
- 涉及目录递归的能力必须补测试，尤其是大目录、空目录、隐藏路径、受保护路径。
- 文件夹分享和大目录打包下载最好一起设计，避免目录枚举和权限逻辑重复。
- 搜索先继续强化元数据检索，再单独规划全文索引，不要把复杂度一次拉满。
