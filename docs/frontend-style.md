# O-Drive 前端视觉风格指南

> 本文档用于记录 O-Drive 当前前端的视觉风格、组件气质和页面重做时应遵守的设计方向。后续重做首页、后台 Tab、分享页或局部卡片时，优先参考本文，避免新页面脱离项目原有风格。

## 设计定位

O-Drive 的界面不是营销型网站，也不是重装饰应用。它更接近一个轻量 SaaS 文件管理器，核心目标是让用户稳定、高效地管理文件、分享链接和后台系统状态。

整体视觉应保持：

- 清爽：浅色背景、白色面板、低噪音层次。
- 克制：少用大面积渐变、夸张阴影、装饰图形和宣传式 hero。
- 工具感：信息清晰，操作可预期，按钮和状态标签明确。
- 精致但不花：用细边框、轻阴影、紧凑间距和状态色建立品质感。
- 可维护：优先复用 token、通用组件和页面已有模式。

一句话概括：**安静、实用、轻量、偏管理工具的云盘产品界面。**

## 三类页面的气质

O-Drive 当前有三类主要前端入口，不同入口的密度和语气应有差异。

### 首页 / 文件管理器

对应入口：

- `public/index.html`
- `public/js/render/home.js`
- `public/css/pages/explorer.css`

首页的核心是文件浏览、搜索、上传、预览和批量操作。它应更像“云盘资源管理器”，不是后台仪表盘。

设计重点：

- 文件卡片和列表是主视觉。
- 路径面包屑、搜索、批量操作要轻而清晰。
- 文件名、大小、修改时间等信息优先保证可读。
- hover 可以有轻微上移、边框加深和柔和阴影。
- 不要把文件区做成过度装饰的内容墙。

推荐密度：

- 文件卡片可以稍微舒展。
- 列表行保持紧凑。
- 主要按钮保持普通尺寸，避免后台式过度压缩。

### 管理后台

对应入口：

- `public/admin.html`
- `public/js/render/pages/admin/*`
- `public/css/pages/admin.css`

后台是高密度运维和配置界面。它的气质应比首页更紧凑、更数据化。

设计重点：

- 使用 12 栅格或明确的两列/三列布局。
- 卡片内部以标题、指标、表格、筛选器、状态标签为主。
- 小字号、小按钮、小标签是合理的。
- 不要做营销页式大标题、大图、大留白。
- 每个模块都要服务明确任务：查看状态、筛选、配置、执行维护动作。

推荐密度：

- 标题通常 `13-18px`。
- 辅助说明通常 `11-13px`。
- 后台小按钮可使用 `24-32px` 高度。
- 表格和列表允许较高信息密度，但必须保证文本不重叠。

### 分享页

对应入口：

- `public/share.html`
- `public/css/pages/share.css`

分享页面向外部访问者，应该更友好、更完整、更像一个独立的小产品界面。

设计重点：

- 居中卡片是主要容器。
- 圆角和空间可以比后台更大。
- 文件名、文件类型、过期/锁定状态要清楚。
- 下载、预览、解锁等主操作要突出。
- 空态、错误态、加载态要更温和。

推荐密度：

- 主卡片可以使用 `20px` 左右圆角。
- 文件预览图标可以更大。
- 按钮高度可保持 `40-44px`，适合触控。

## 颜色系统

颜色来源以 `public/css/tokens.css` 为准。新增样式应优先使用变量，不要散落硬编码颜色。

### 基础颜色

| 用途 | 变量 | 当前浅色值 | 使用建议 |
| --- | --- | --- | --- |
| 页面背景 | `--bg` | `#f1f5f9` | 页面底色，不用于卡片内部强调。 |
| 主面板 | `--panel` | `#ffffff` | 卡片、弹窗、表格主背景。 |
| 次级面板 | `--panel-soft` | `#f8fafc` | 卡片内块、筛选条、浅 hover 区域。 |
| 强面板 | `--panel-strong` | `#ffffff` | 需要更明确浮层时使用。 |
| 细边框 | `--line` | `rgba(148, 163, 184, 0.5)` | 默认边框。 |
| 强边框 | `--line-strong` | `rgba(100, 116, 139, 0.5)` | hover、active 或分隔强调。 |
| 主文本 | `--text` | `#1e293b` | 标题、正文、重要数字。 |
| 次文本 | `--muted` | `#94a3b8` | 辅助说明、标签说明、空态说明。 |

### 品牌色和状态色

| 用途 | 变量 | 当前浅色值 | 使用建议 |
| --- | --- | --- | --- |
| 主色 | `--accent` / `--primary` | `#0e7490` | 主按钮、链接、选中态、焦点。 |
| 主色增强 | `--accent-strong` | `#0f4c5c` | hover、重点文字。 |
| 主色浅底 | `--accent-soft` | `rgba(14, 116, 144, 0.1)` | 选中背景、信息块浅色背景。 |
| 成功 | `--success` | `#18794e` | 已启用、成功、在线。 |
| 警告 | `--warning` | `#b76e11` | 即将过期、需要注意。 |
| 危险 | `--danger` | `#c0392b` | 删除、失败、错误。 |

### 色彩使用原则

- 主色青蓝只用于操作、选中和关键状态，不要大面积铺满。
- 成功、警告、危险色应以浅底加文字为主，少用实心大色块。
- 后台卡片不要做一整屏同色渐变。
- 彩色标签可以出现多个，但同一模块内不要超过 3-4 个主状态色。
- 代码、路径、URL、token 等技术文本可用 `--accent` + `--accent-soft`。

### 暗色模式

暗色模式由 `[data-theme="dark"]` 下的 token 支持。新增 UI 时：

- 必须使用 token 变量，避免只适配浅色。
- 不要写死白底、黑字、浅灰边框。
- 图片和 SVG 图标要确认暗色下不会过亮。
- `box-shadow` 在暗色下应更克制，优先靠边框和背景层次。

## 字体与排版

项目使用系统字体栈：

```css
--body-font: -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
```

排版特点：

- 字体是系统 UI 字体，不追求品牌展示字体。
- 标题使用 600-700 字重。
- 辅助文字使用 `--muted`，字号更小。
- 数据和路径常使用等宽字体。

推荐字号：

| 场景 | 字号 | 字重 | 说明 |
| --- | --- | --- | --- |
| 页面主标题 | `18-28px` | `700` | 首页/分享页可大一些，后台少用大标题。 |
| 后台卡片标题 | `13-15px` | `600-700` | 保持紧凑。 |
| 正文/按钮 | `13-15px` | `500-600` | 普通按钮不宜太轻。 |
| 后台辅助文字 | `11-13px` | `400-600` | 用于说明、标签、meta。 |
| 表格文字 | `11-13px` | `400-600` | 数据密集区域。 |
| 指标数字 | `20-28px` | `700` | 使用 `font-variant-numeric: tabular-nums`。 |

排版规则：

- 后台卡片内不要使用 hero 级大字。
- 长文件名、路径、URL 必须考虑截断、换行或横向滚动。
- 路径和 URL 优先 `word-break: break-all`。
- 表格、数字、容量、百分比适合使用 `tabular-nums`。
- 不使用负字距；保持默认 letter spacing，只有 uppercase 小标签可加 `0.05-0.08em`。

## 空间、圆角和边框

项目整体靠细边框和浅背景建立层级。

### 圆角

来源：

- `--radius-xl: 16px`
- `--radius-lg: 14px`
- `--radius-md: 12px`
- `--radius-sm: 8px`

使用建议：

- 通用卡片：`10-12px`
- 后台小卡片和信息块：`6-10px`
- 按钮：`6-10px`
- 标签：`3-8px` 或胶囊形 `999px`
- 分享页主卡片：可使用 `20px`
- 弹窗：`16px`

不要把后台所有元素都做成大圆角。后台应该更稳、更工具化。

### 边框

默认使用：

```css
border: 1px solid var(--line);
```

hover 或选中使用：

```css
border-color: var(--line-strong);
```

选中态可以叠加：

```css
box-shadow: 0 0 0 2px var(--accent-soft);
```

### 阴影

项目阴影非常轻：

- 普通卡片：常常没有明显阴影，只靠边框。
- hover 卡片：`0 2px 8px rgba(0, 0, 0, 0.04-0.06)`。
- 分享页主卡片和下拉浮层可更明显一些。
- 不使用重阴影营造“悬浮卡片堆叠”。

## 布局语言

### 总体原则

- 页面优先使用 flex/grid，不写固定高度魔法数。
- 文件区、后台页等主要区域要保证 `min-height: 0`，避免滚动容器失效。
- 后台布局以栅格、表格、列表、状态卡片为主。
- 分享页以单卡片居中为主。
- 局部组件可以紧凑，但不能靠裁切隐藏信息。

### 后台布局

后台已有一套 `ap-*` 和 `ov-*` 风格：

- `ap-grid` 使用 12 栅格。
- `ap-card` 是基础后台卡片。
- `ap-card-head` 和 `ap-card-body` 控制头部和内容。
- `ov-*` 多用于具体 Tab 的业务模块。

后台新增模块时：

1. 先判断能否使用现有卡片、表格、按钮。
2. 单个 Tab 的样式放在 `public/css/pages/admin.css`。
3. 不要在 Tab 子渲染器里重建外层 `.explorer-card`。
4. 不要让单个模块撑破整体后台容器。
5. 高度不足时优先压缩内部信息结构，再考虑局部滚动。

### 首页文件布局

文件卡片风格：

- 白色卡片。
- 细边框。
- 文件图标居中。
- 文件名截断。
- hover 轻微上移。
- 选中态使用主色边框和浅主色外发光。

文件列表风格：

- 更注重扫描效率。
- 文件名、路径、时间、大小信息要对齐。
- hover 背景应轻，不要破坏表格可读性。

### 分享页布局

分享页使用三段式卡片：

- 顶部：品牌和状态。
- 中部：文件图标、文件名、说明。
- 底部：操作按钮和元信息。

分享页可以比后台更有“完成度”，但仍然不要变成营销落地页。

## 核心组件规范

### 按钮

通用按钮：

- 使用 `.btn`、`.btn-primary`、`.btn-ghost`、`.btn-danger`、`.btn-small`。
- 高度通常 `38px`。
- 圆角通常 `8-10px`。
- hover 改变背景和边框。
- active 可轻微缩放。

后台按钮：

- 可以更紧凑，如 `30px`、`24px`。
- 文字短，动作明确。
- 删除、清理、危险操作使用 danger 样式。
- 图标按钮要有 `aria-label`。

按钮文案建议：

- 使用动词：复制、刷新、保存、删除、重试。
- 避免过长说明写进按钮。
- 同一组按钮主次要明确。

### 输入框和表单

通用输入框：

- 背景 `var(--panel-soft)` 或 `var(--panel)`。
- 边框 `var(--line)`。
- focus 使用青蓝边框和柔和外发光。
- 圆角 `8px` 左右。

后台表单：

- 可压缩高度和间距。
- 多个短字段可以并排。
- 开关/复选框使用紧凑块状布局时，要保证标签可见。
- 不要让辅助说明挤占主要输入空间；必要时隐藏或移到 tooltip/帮助文案。

### 卡片

基础卡片：

```css
background: var(--panel);
border: 1px solid var(--line);
border-radius: 10px 或 12px;
```

卡片内部常见结构：

- 头部：标题 + 状态/操作。
- 主体：数据、列表、表格、设置。
- 辅助区：小标签、说明、更新时间。

不要：

- 卡片套卡片过深。
- 大面积渐变卡片。
- 把页面 section 全都做成浮动大卡片。
- 用装饰元素填补空白。

### 标签和状态

项目大量使用 soft badge：

- 背景是状态色 `0.08-0.12` 透明度。
- 文字使用对应状态色。
- 圆角小或胶囊形。
- 字号 `10-12px`。
- 字重 `600`。

适合标签化的信息：

- 已启用 / 未配置。
- Active / Expired / Error。
- DAV Level 1。
- 文件类型。
- 权限和限制。

### 表格和列表

后台表格要求：

- 表头小字号、muted、uppercase 或半强调。
- 行 hover 使用 `var(--panel-soft)`。
- 单元格 padding 通常 `7-12px`。
- 长路径、token、URL 用等宽字体并截断。
- 操作按钮靠右或放最后列。

列表要求：

- 左侧主信息，右侧状态/操作。
- 主要文本加粗。
- meta 信息 muted。
- 重要状态可使用左边框色或小圆点。

### 空态

空态结构：

- `empty-orb` 图标容器。
- `empty-title` 标题。
- `empty-copy` 说明。

后台空态应紧凑，不要占据过多高度。分享页空态可以更居中、更舒展。

### 弹窗和浮层

弹窗：

- 居中。
- 背景遮罩 `rgba(0,0,0,0.25)`。
- 卡片 `16px` 圆角。
- 内容宽度一般 `460px`，预览弹窗可更宽。

浮层：

- 使用 `var(--panel)`。
- 边框 `var(--line)`。
- 阴影可以比普通卡片略明显。
- 动效短，避免复杂展开动画。

## 图标和文件类型

项目有 `public/icons/` 文件类型图标和 `public/js/ui/icons.js` 通用图标。

使用原则：

- 文件类型优先使用已有图标资产。
- 按钮内图标要服务动作，不要纯装饰。
- 后台状态可以使用小圆点、badge、进度条，不一定需要图标。
- 分享页文件预览图标可以更大、更友好。
- 暗色模式下注意图标亮度，已有 CSS 对部分图标做了降亮处理。

## 动效和交互状态

项目动效应轻而短：

- hover：`120-160ms`
- 页面/弹窗进入：`180-220ms`
- 背景主题切换：`280ms`
- 进度条变化：`300-400ms`

常见状态：

- hover：背景变浅、边框变强、轻阴影。
- active：按钮轻微缩放。
- focus：青蓝外发光。
- selected：主色边框 + 浅主色背景或外发光。
- disabled：降低透明度，移除 pointer events。

不要使用：

- 大幅弹跳。
- 长时间过渡。
- 频繁闪烁。
- 与数据状态无关的装饰动画。

## 响应式规则

项目主要断点：

- `1024px`
- `980px`
- `768px`
- `720px`
- `480px`

响应式原则：

- 后台多列布局在窄屏变单列。
- Tab bar 可横向滚动，不要挤压到不可读。
- 文件网格在小屏可变成单列。
- 表格在小屏优先横向滚动，或隐藏次要列。
- 弹窗在小屏减小 padding，输入框保持触控高度。
- 按钮组在小屏允许换行或占满宽度。

移动端尤其要检查：

- 按钮文字是否溢出。
- 文件名是否遮挡 meta。
- 卡片内部是否出现双重滚动。
- 表格是否能横向滚动。
- 复制类 URL 是否会撑破容器。

## 文案风格

O-Drive 的界面文案应简短、明确、偏工具语气。

推荐：

- 保存规则
- 复制地址
- 刷新状态
- 暂无访问控制规则
- 使用管理员密码
- 单次 PUT <= 100MB

避免：

- 过度解释功能原理。
- 在卡片内写教程式长段落。
- 使用营销话术。
- 把键盘快捷键、设计说明写到界面上。

后台可以更简短，分享页可以更温和。

## 页面重做时的工作流

重做某个页面或卡片时，建议按以下顺序：

1. 明确页面类型：文件管理、后台配置、分享访问。
2. 确定密度：后台紧凑，分享页舒展，首页居中。
3. 先复用现有 token 和组件。
4. 修改源 CSS，不直接只改构建产物。
5. 保持外层布局不变，优先调整内部信息结构。
6. 检查浅色和暗色模式。
7. 检查桌面和移动端。
8. 运行 `npm run build` 生成 CSS 产物。
9. 涉及交互时运行 `npm test`。

## 常见场景设计建议

### 配置信息卡片

适合结构：

- 左侧：核心配置值，例如 URL、路径、token、阈值。
- 右侧：状态、限制、权限、更新时间。
- 下方：少量操作按钮。

不要把所有字段纵向堆叠到很高，后台卡片尤其应避免底部大空白。

### 服务状态面板

适合结构：

- 顶部：服务名 + 状态 badge。
- 主体：连接方式或关键指标。
- 摘要：协议、限制、能力、健康状态。

WebDAV、Webhook、维护任务等都适合这种结构。

### 规则编辑表单

适合结构：

- 路径/名称等主字段放前面。
- 开关项靠近相关字段。
- 密码、备注等次要字段可并排。
- 保存按钮固定在标题区或表单末尾，但不要重复。

如果一屏显示不完，优先重新组织信息，而不是马上加滚动。

### 分享列表项

适合结构：

- 左侧：分享名称/路径/状态标签。
- 中间：访问次数、过期时间、权限等指标。
- 右侧：复制、删除等操作。

过期、耗尽、即将过期可用左边框或状态色标记。

### 操作日志和任务列表

适合结构：

- 表格或紧凑列表。
- 时间、动作、路径、结果清楚分列。
- 错误详情用 muted 或 danger 文本。
- 重试按钮靠近失败状态。

## 不推荐的方向

后续页面重做时，应避免：

- 大面积紫色、蓝紫渐变。
- 营销站式 hero 区。
- 过度玻璃拟态。
- 大量装饰圆点、光斑、背景纹理。
- 一屏内套太多卡片。
- 后台页面使用过大字号和过大留白。
- 使用单一颜色变体撑完整页面。
- 临时硬编码颜色而不使用 token。
- 只改 `public/admin.css` 等构建产物，不改源 CSS。
- 为一个局部按钮新增一套全局按钮系统。

## 设计验收清单

提交前可以快速检查：

- 是否符合页面类型的密度。
- 是否使用 `tokens.css` 中的颜色和圆角变量。
- 是否兼容暗色模式。
- 文字是否不会溢出、重叠、遮挡。
- 长路径、URL、文件名是否处理了截断或换行。
- hover、focus、disabled 状态是否完整。
- 移动端是否从多列变单列或可横向滚动。
- 后台页面是否避免了大面积空白。
- 是否修改了源 CSS 并重新构建产物。
- 是否没有引入新的前端框架或第二套 UI 范式。

## 当前风格摘要

如果只记一版最短规则：

- 颜色：浅灰背景、白色面板、青蓝主色、低饱和状态色。
- 形状：细边框、轻阴影、`8-12px` 圆角。
- 排版：系统字体，后台小字号高密度，分享页更舒展。
- 组件：卡片、表格、标签、紧凑按钮、轻量空态。
- 交互：短动效、浅 hover、青蓝 focus、清晰 disabled。
- 方向：像可靠的云盘工具，不像营销页或重装饰作品。
