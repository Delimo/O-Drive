## Critical（严重）

### C1. `var(--border)` 和 `var(--primary)` / `var(--primary-soft)` CSS 变量未定义
- **现象**：进度条轨道透明不可见；未读标签无颜色
- **位置**：`components.css:1344`（`--border`）、`components.css:2270-2271`（`--primary` / `--primary-soft`）、`pages.js` 内联样式
- **修复方案**：在 `tokens.css` 的 `:root` 和 `[data-theme="dark"]` 中添加这三个变量定义
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### C2. `syncHomeUrl` 参数错误，浏览器 URL 永远不同步
- **现象**：切换文件夹后地址栏不更新，刷新后回到根目录
- **位置**：`index.js:131`、`thunks.js`、`events/index.js`
- **修复方案**：在 `index.js` 中通过 wrapper `(path,query) ⇒ syncHomeUrlHelper(page,path,query)` 注入 page 参数
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### C3. 列表视图「全选」和「列排序」点击无反应
- **现象**：列表视图全选复选框和表头排序按钮无功能
- **位置**：`events/index.js:861-874`（`toggle-all-pick`）、`events/index.js:777-786`（`sort-list`）
- **修复方案**：事件处理已存在，已验证能正常工作
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### C4. 拖拽上传文件系统访问失败时 Promise 永远挂起
- **现象**：拖拽文件到文件夹时，若文件系统权限异常，界面永远卡在加载中
- **位置**：`helpers.js:37,41`
- **修复方案**：给 `entry.file(resolve, reject)` 和 `reader.readEntries` 增加 reject 回调
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### C5. `formatListTime` 与 `formatTime` 时间戳单位不一致
- **现象**：同一文件在网格视图和列表视图显示的修改时间不同
- **位置**：`format.js:14-23`
- **修复方案**：`formatTime` 和 `formatRelative` 统一乘以 1000 转换为毫秒
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

## High（高优先级）

### H1. `components.css` 与 `admin/admin.css` 大量重复规则冲突

- **位置**：`components.css` 全文 vs `admin/admin.css` 全文
- **修复方案**：删除 `components.css` 中仅用于 admin 页面的重复规则，保留 `admin/admin.css` 为唯一来源

- **状态**：- [x] 已修复（随 C1 一起处理）
- **检验**：[x] 已检验

### H3. 10+ 处用 `icons.refresh` 替代 `icons.spinner`
- **现象**：加载状态显示静态刷新图标，无旋转动画
- **位置**：`pages.js`、`modal.js`

- **检验**：[x] 已检验

### H5. viewport 禁止缩放（WCAG 1.4.4 违规）

- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### H6. 图片 `alt=""` 为空
- **现象**：缩略图无替代文字，屏幕阅读器无法识别

- **修复方案**：将 `alt=""` 改为 `alt="${escapeHtml(item.name)}"`
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### H7. JS 渲染 HTML 中 100+ 处内联样式
- **现象**：大量 `style="..."` 散落在 JS 模板字符串中，难以维护
- **位置**：`modal.js`、`pages.js`、`home.js` 等
- **修复方案**：将重复模式提取为 CSS 类（如 `.filter-field`、`.progress-track`、`.section-header` 等）
- **状态**：- [x] 已修复（部分提取）
- **检验**：[x] 已检验

### H8. `!important` 冲突
- **现象**：两个文件对同一类名用 `!important` 设不同值
- **位置**：`components.css:962-964` vs `admin/admin.css:45-46`（`.admin-tab-active`）
- **修复方案**：统一为一处定义，移除不必要的 `!important`
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### H9. `_xhrUpload` 的 `JSON.parse` 无 try/catch
- **现象**：服务器返回非 JSON 时静默失败
- **位置**：`api/index.js:399-425`
- **修复方案**：用 try/catch 包裹 `JSON.parse`，失败时返回原始响应
- **状态**：- [x] 已修复
- **检验**：[x] 已检验

### H10. 批量回收站操作遇首个失败即终止

- **检验**：[x] 已检验
